# GStack-Inspired Staff Review

Source: GStack by Garry Tan, MIT licensed, https://github.com/garrytan/gstack.

Use after implementation and before release.

## Review Focus

- Correctness and regressions.
- Security and authorization.
- Data handling and privacy.
- Operational risk.
- Test gaps.
- Unnecessary complexity.
- Compatibility with existing patterns.

Lead with blocking findings. If there are no blockers, say so clearly and
record residual risk.
