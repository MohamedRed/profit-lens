# GStack-Inspired Engineering Plan Review

Source: GStack by Garry Tan, MIT licensed, https://github.com/garrytan/gstack.

Use before coding meaningful engineering work.

## Required Review

- Scope and non-goals.
- Existing system constraints.
- Files or services likely affected.
- Migration or compatibility risk.
- Security, privacy, and data impact.
- Test plan.
- Rollback plan if production is involved.

Return approval-needed items separately from implementation-ready items.
